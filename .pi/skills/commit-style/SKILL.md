# Commit Style

## Правило

Сообщения коммитов должны быть написаны **на русском языке** и отвечать на вопрос **«Что сделано?»**.

Предпочтительная форма — краткое описание в **прошедшем времени**:

- `Добавлена валидация форм`
- `Обновлена зависимость Newtonsoft.Json`
- `Исправлена ошибка при пустом вводе`
- `Переработана архитектура модуля авторизации`
- `Удалён устаревший код`

### ❌ Неправильно
- `Add validation`
- `fix bug`
- `Обновить зависимости` (повелительное наклонение)
- `WIP`

### ✅ Правильно
- `Добавлена валидация форм`
- `Исправлена ошибка 500 при сохранении`
- `Обновлены пакеты до .NET 9`

## Использование

Скилл автоматически применяется, если он указан в `skills` секции `CLAUDE.md` проекта в любом из `.pi/agents/*.md`.

```yaml
# .pi/agents/my-agent.md
skills:
  - commit-style
```

Или в `CLAUDE.md`:

```markdown
Использовать скилл commit-style для коммитов.
```
